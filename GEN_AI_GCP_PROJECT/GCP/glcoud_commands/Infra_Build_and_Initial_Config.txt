
# Cloud Run & GCP Setup Documentation

## Introduction
This document provides a detailed guide on setting up and configuring various Google Cloud Platform (GCP) services and deploying them with Cloud Run. 
It covers enabling required APIs, creating service accounts, setting roles for users, creating necessary resources like Artifact Registry, SQL instances, 
Redis DB, and deploying a Cloud Run service. The guide is designed for users working on the backoffice-optimization-dev project and aims to streamline 
the GCP setup and deployment process for GitLab Pipelines and Cloud Run services.

## Table of Contents

1. **Enable APIs**
   - This section describes how to enable necessary GCP APIs for the project.
   
2. **Add Service Accounts for GitLab Pipelines and Cloud Run Deployment**
   - Details the process of creating and assigning service accounts with required roles for both GitLab Pipelines and Cloud Run services.

3. **Add Roles to Specific Users for GCP Services Management**
   - Describes how to grant necessary roles to GlobalLogic users for managing and editing GCP services.

4. **Create Artifact Registry**
   - Instructions for creating an Artifact Registry to store Docker images used in the deployment pipeline.

5. **Create SQL Instance**
   - Provides steps for creating a Cloud SQL instance with PostgreSQL for the project.

6. **Create PostgreSQL User and Database**
   - Describes how to create users and databases in the PostgreSQL instance for DEV and PROD environments.

7. **Enable the pgvector Extension**
   - Instructions for enabling and verifying the pgvector extension on the PostgreSQL instance.

8. **Create Two Private Buckets**
   - Guide on creating private buckets in GCP for storing project data securely.

9. **Create Redis DB Instance**
   - Steps to create a Redis instance on GCP for caching purposes.

10. **Cloud Run Deployment Command**
    - Final section outlining the command to deploy the application to Cloud Run, including environment variables, database integration, and memory/cpu settings.

11. **Requirements**
    - Ensures all requirements are met for a successful deployment, including permissions for Cloud Run service accounts.

Create SQL instance
===================

gcloud sql instances create <instance name> \
  --project=YOUR_PROJECT_ID \
  --database-version=POSTGRES_17 \
  --edition=enterprise \
  --cpu=2 \
  --memory=8192MB \
  --region=us-central1 \
  --network=projects/YOUR_PROJECT_ID/global/networks/default \
  --no-assign-ip \
  --enable-private-path-for-google-cloud-services \
  --root-password=test \
  --availability-type=regional \
  --storage-type=SSD \
  --storage-size=20 \
  --storage-auto-increase \
  --backup-start-time=02:00 \
  --enable-backup \
  --retained-backups=7 \
  --retention-unit=TIME

Create two buckets:
===================
Steps to Create Private Buckets Using gsutil
​Ensure you're authenticated and have set the correct project:​

From GCLOUD SDK CLI:
gcloud auth login
gcloud config set project backoffice-optimization-dev
​
Create the genai-vacancies-dev bucket:​
gsutil mb -c STANDARD -l us-central1 gs://genai-vacancies-dev/

​Create the genai-candidates-dev bucket:​
gsutil mb -c STANDARD -l us-central1 gs://genai-candidates-dev/

​Enable uniform bucket-level access to make the buckets private:​
gsutil uniformbucketlevelaccess set on gs://genai-vacancies-dev/
gsutil uniformbucketlevelaccess set on gs://genai-candidates-dev/

REDIS DB instance:
==================

gcloud redis instances create langgraph-redis \
  --region=us-central1 \
  --tier=basic \
  --memory-size=16 \
  --redis-version=redis_7_0 \
  --network=projects/backoffice-optimization-dev/global/networks/default


Parameters:

--size: Capacity in GB (1–300 GB)
--tier: BASIC or STANDARD_HA (high availability)
--redis-version: e.g., redis_7_0
--network: VPC network (default or custom)

Cloud Run Deployment Command
============================

gcloud run deploy langgraph-service \
  --project=YOUR_PROJECT_ID \
  --image=us-central1-docker.pkg.dev/backoffice-optimization-dev/glia/glia:latest \
  --region=us-central1 \
  --platform=managed \
  --vpc-egress=all \
  --add-cloudsql-instances=YOUR_PROJECT_ID:us-central1:vector-db-instance \
  --set-env-vars="DB_HOST=/cloudsql/YOUR_PROJECT_ID:us-central1:vector-db-instance,DB_USER=postgres,DB_PASSWORD=test,DB_NAME=vector_db" \
  --memory=512Mi \
  --cpu=1 \
  --allow-unauthenticated
