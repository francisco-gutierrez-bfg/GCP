Docker build and push commands:
-------------------------------
You can create images locally and push them, to do so, please follow the below example:

1- Loign your gcloud cli:
   gloud auth login

2- Create a Dockerfile in your system.

3- Build the image:
   Important: The image must have the name of your repo as follows
   Docker build command sample:
    docker build -t europe-west9-docker.pkg.dev/olb-beneficiaries-api-qa/ms-appdata

4- Push the image:
   Docker push command sample: 
    docker push europe-west9-docker.pkg.dev/olb-beneficiaries-api-qa/ms-appdata:latest


GAR (Artifact Registry Repository):
-----------------------------------
GAR must be enabled and ready to create repos.

To create a repo:

Repo name: ms-appdata
Region: europe-west9
Encryption; From Google
Format: Docker
Mode: Standard
Location type: Region
Inmutable Image tags: Disabled
Clean policy; Test
Vulnerability scann: disabled

gcloud artifacts repositories create ms-appdata \
  --project=olb-beneficiaries-api-qa \
  --location=europe-west9 \
  --repository-format=docker \
  --description="Docker repository for ms-appdata" \
  --encryption-key="google-managed" \
  --mode=standard \
  --location-type=region \
  --immutable-tags=false \
  --vulnerability-scan=disabled

Cloud Run Deployment:
---------------------

GitHub should deploy the image into GAR repository (Google Artifact Registry)

gcloud run deploy ms-appdata \
          --image europe-west9-docker.pkg.dev/olb-beneficiaries-api-qa/ms-appdata/ms-appdata:latest \
          --platform managed \
          --service-account clourun-olenbee@olb-beneficiaries-api-qa.iam.gserviceaccount.com \
          --region europe-west9 \
          --vpc-connector projects/olb-beneficiaries-api-qa/locations/europe-west9/connectors/olb-dev-network-glb-netspoke \
          --vpc-egress all-traffic \
          --allow-unauthenticated
