Docker build and push commands:
-------------------------------
You can create images locally and push them, to do so, please follow the below example:

1- Loign your gcloud cli:
   gloud auth login

2- Create a Dockerfile in your system.

3- Build the image:
   Important: The image must have the name of your repo as follows
   Docker build command sample:
    docker build -t europe-west9-docker.pkg.dev/olb-backoffice-api-qa-backnd/backoffice

4- Push the image:
   Docker push command sample: 
    docker push europe-west9-docker.pkg.dev/olb-backoffice-api-qa-backnd/backoffice:latest


GAR (Artifact Registry Repository):
-----------------------------------
GAR must be enabled and ready to create repos.

To create a repo:

Repo name: backoffice
Region: europe-west9
Encryption; From Google
Format: Docker
Mode: Standard
Location type: Region
Inmutable Image tags: Disabled
Clean policy; Test
Vulnerability scann: disabled

gcloud artifacts repositories create backoffice \
  --project=olb-backoffice-api-qa-backnd \
  --location=europe-west9 \
  --repository-format=docker \
  --description="Docker repository for backoffice" \
  --encryption-key="google-managed" \
  --mode=standard \
  --location-type=region \
  --immutable-tags=false \
  --vulnerability-scan=disabled

Cloud Run Deployment:
---------------------

GitHub should deploy the image into GAR repository (Google Artifact Registry)

gcloud run deploy backoffice \
    --image europe-west9-docker.pkg.dev/olb-backoffice-api-qa-backnd/backoffice/backoffice@sha256:b35572a6cbebd86074582c40dbbeac147056941a5fee1d92f2a17065d9aa051d \
    --region europe-west9 \
    --platform managed \
    --service-account clourun-olenbee@olb-backoffice-api-qa-backnd.iam.gserviceaccount.com \
    --vpc-connector projects/olb-backoffice-api-qa-backnd/locations/europe-west9/connectors/olb-dev-network-glb-netspoke \
    --vpc-egress all-traffic \
    --allow-unauthenticated
