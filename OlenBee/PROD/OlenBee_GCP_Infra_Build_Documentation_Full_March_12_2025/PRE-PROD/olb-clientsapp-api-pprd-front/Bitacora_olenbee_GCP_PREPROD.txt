Docker build and push commands:
-------------------------------
You can create images locally and push them, to do so, please follow the below example:

1- Loign your gcloud cli:
   gloud auth login

2- Create a Dockerfile in your system.

3- Build the image:
   Important: The image must have the name of your repo as follows
   Docker build command sample:
    docker build -t europe-west9-docker.pkg.dev/olb-backoffice-api-pprd-front/clientsapp/clientsapp

4- Push the image:
   Docker push command sample: 
    docker push europe-west9-docker.pkg.dev/olb-backoffice-api-pprd-front/clientsapp/clientsapp:latest


GAR (Artifact Registry Repository):
-----------------------------------
GAR must be enabled and ready to create repos.

To create a repo:

Repo name: backofficefront
Region: europe-west9
Encryption; From Google
Format: Docker
Mode: Standard
Location type: Region
Inmutable Image tags: Disabled
Clean policy; Test
Vulnerability scann: disabled

gcloud artifacts repositories create clientsappfront \
  --project=olb-backoffice-api-pprd-frontend \
  --location=europe-west9 \
  --repository-format=docker \
  --description="Docker repository for clientsapp front" \
  --encryption-key="google-managed" \
  --mode=standard \
  --location-type=region \
  --immutable-tags=false \
  --vulnerability-scan=disabled

   --image europe-west9-docker.pkg.dev/olb-backoffice-api-pprd-front/clientsappfront/clientsappfront@sha256:4f1dd64c14b1e780a581e5704c697bf4d18a09ebfcb39929346b3219b22881d6 \

   --platform managed \
   --service-account clourun-olenbee@olb-backoffice-front-prod.iam.gserviceaccount.com \

   --vpc-egress all-traffic \
   --allow-unauthenticated
