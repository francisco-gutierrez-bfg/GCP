Configurar componentes para el Load Balancer:
---------------------------------------------
gcloud compute backend-services create clientsappfront \
    --global \
    --load-balancing-scheme EXTERNAL_MANAGED

gcloud compute network-endpoint-groups create clientsappfront-neg \
    --region=europe-west9 \
    --network-endpoint-type=SERVERLESS \
    --cloud-run-service=clientsapp-front

gcloud compute backend-services create clientsappfront-backend \
    --global \
    --load-balancing-scheme=EXTERNAL_MANAGED

gcloud compute backend-services add-backend clientsappfront-backend \
    --global \
    --network-endpoint-group=clientsappfront-neg \
    --network-endpoint-group-region=europe-west9

Crear el Load Balancer:
-----------------------

Crear el URL Map
Este mapa de URLs dirige el tráfico al backend configurado:

gcloud compute url-maps create clientsappfront-url-map \
    --default-service=clientsappfront-backend

Crear el Proxy HTTP
Se configura un proxy HTTP, sin cifrado SSL:

gcloud compute target-http-proxies create clientsappfront-http-proxy \
    --url-map=clientsappfront-url-map

Crear una IP estática global
Reservar una IP para el Load Balancer:

gcloud compute addresses create clientsappfront-ip \
    --global

Obtener la IP asignada:

gcloud compute addresses describe clientsappfront-ip --global --format="get(address)"


Crear la Forwarding Rule para HTTP
Configurar la regla de reenvío para el puerto 80:

gcloud compute forwarding-rules create clientsappfront-forwarding-rule \
    --global \
    --target-http-proxy=clientsappfront-http-proxy \
    --ports=80 \
    --address=clientsappfront-ip
